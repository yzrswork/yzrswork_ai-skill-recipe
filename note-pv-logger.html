📊 note PV Logger

— note のアクセス数（ビュー）を全記事まとめて書き出す

note のダッシュボードに表示される記事ごとのビュー・スキ・コメント数を全件まとめて取得し、 CSV / TSV / Markdown で書き出すユーザースクリプトです。 note にログインしているブラウザのセッションをそのまま使うので、 API キーやトークンの登録は不要。スプレッドシートでの集計や、Obsidian でのビュー数の記録に使えます。

note にログインした状態で note.com を開く │ ▼ 右下のボタンを押す ──→ パネルで期間を選んで「取得」 │ （全記事をページめくりして自動収集） ▼ CSV / TSV / Markdown で保存・コピー ├─ CSV → Excel・スプレッドシートに読み込み ├─ TSV → そのまま表に貼り付け └─ .md → Obsidian Vault に置いて記録（日付つきスナップショット）

012種類のスクリプト

用途に合わせて好きな方（または両方）を入れてください。取得する数値はどちらも同じで、書き出す形式だけが違います。

スクリプトボタン書き出せる形式主な用途
note-pv-logger.user.js
(PV Logger)		 📊 PV CSV（Excel用・BOM付き）
TSV（コピー）		 スプレッドシートで集計・グラフ化したいとき
note-pv-md-logger.user.js
(PV Markdown Logger)		 📝 PV→MD Markdown（.md保存・コピー） Obsidian Vault にビュー数を記録したいとき

02仕組みと特徴

  • note のダッシュボードが内部で使っている統計 API（/api/v1/stats/pv）を、最終ページまで自動でめくって全記事ぶん取得します。
  • 認証はログイン中のブラウザのセッション（Cookie）をそのまま使います。トークンやパスワードの入力は一切不要です。
  • 期間（全期間 / 月間 / 週間）を選んで取得できます。
  • 取得結果はパネル内の表で確認できるほか、ボタン一発で保存・コピーできます。
  • 取得日時を一緒に記録するので、定期的に取ればビュー数の推移も追えます。
⚠️ 自分のアカウントのデータ取得にだけ使ってください これは note の非公式な内部 API を利用する方法です。自分のログイン中アカウントの統計を読み取るだけですが、 note 側の仕様変更で動かなくなる可能性があります。自己責任でご利用ください。

03PC への導入（5分）

  1. ユーザースクリプトマネージャーを入れる： Tampermonkey または Violentmonkey
  2. Chrome の場合のみ：デベロッパーモードを有効化する
    アドレスバーに chrome://extensions と入力 → 右上の「デベロッパー モード」をオンにする。
    これをしないと Chrome がスクリプトのインストールをブロックします（「このウェブサイトから追加することはできません」エラー）。Firefox や Violentmonkey ではこの手順は不要です。
  3. 下のボタンから、使いたい方のスクリプトをインストール（両方入れても構いません）
⚠️ Chrome で「このウェブサイトから追加することはできません」と出る場合 次の順に確認してください：
  1. chrome://extensions → 右上「デベロッパー モード」をオン
  2. Chrome を完全に再起動（デベロッパーモードはブラウザ再起動後に反映されます）
  3. このページを file://（ローカルファイル）で開いている場合：chrome://extensions → Tampermonkey の「詳細」→「ファイルの URL へのアクセスを許可する」をオン
それでもダメなら手動インストールが確実です：Tampermonkey アイコン → ダッシュボード → 「ユーティリティ」タブ → 「ファイルからインポート」で .user.js ファイルを選択。

04iPhone への導入

iPhone でも Safari 上の note.com で同じように使えます。アップローダーなどの追加スクリプトは不要、1本入れるだけです。

  1. App Store で Userscripts アプリを入れ、設定 → Safari → 機能拡張 で有効化
  2. Safari で本ページを開き、上のボタンから使いたい方のスクリプトをインストール
  3. Safari で note.com にログインした状態で開き、右下のボタンから取得
取得した .md / CSV はどこへ行く？ iPhone ではダウンロードした .md や .csv は「ファイル」アプリに保存されます。 Obsidian の Vault フォルダ（iCloud など）に移動すれば取り込めます。表に貼るだけなら「コピー」が手軽です。

05使い方

  1. note にログインした状態で note.com の任意のページを開く
  2. 画面右下に出るボタン（📊 PV または 📝 PV→MD）をタップしてパネルを開く
  3. 期間（全期間 / 月間 / 週間）を選んで「取得」を押す
  4. 記事数と合計ビューが表示され、表で内容を確認できる
  5. あとは書き出すだけ：
    • CSV/TSV版：「CSV保存」（Excel向け）／「コピー」（スプレッドシートに貼り付け）
    • Markdown版：「.md保存」（Obsidian に置く）／「MDコピー」
ログインしていないと取れません 数値が取れない・「認証エラー」と出る場合は、まず同じブラウザで note にログインしているか確認してください。

06書き出されるデータ

CSV / TSV版

1行 = 1記事。取得日時の列が入るので、日付ごとに同じシートへ貯めていけば推移を追えます。

取得日時,期間,記事ID,タイトル,URL,ビュー,スキ,コメント
2026-06-13 21:40,all,1234567890,記事タイトル,https://note.com/yzrswork/n/xxxx,1234,56,7
...

Markdown版（Obsidian向け）

取得するたびに note-pv_<期間>_<日時>.md という1スナップショット = 1ファイルが作られます。 フロントマターに合計値が入るので、Obsidian の Dataview で推移を集計できます。

note-pv_all_20260613-2140.md

---
type: note-pv-snapshot
fetched_at: "2026-06-13 21:40"
date: 2026-06-13
filter: all
filter_label: "全期間"
articles: 42
total_pv: 12345
total_like: 678
total_comment: 9
tags: [note, pv, stats]
---

07注意事項

項目内容
非公式 APInote の内部 API を使う非公式な方法です。note 側の仕様変更で、ある日突然動かなくなる可能性があります。
自分のデータのみ取得できるのは、ログイン中のあなた自身のアカウントの統計だけです。
トークン不要ブラウザのログインセッションを使うため、PAT やパスワードの保存はありません。
取得件数が多い場合記事数に応じて API を複数回呼びます（負荷軽減のため各ページの間に短い待ちを入れています）。記事が非常に多いと取得に少し時間がかかります。
数値の基準表示される数値は note のダッシュボードと同じ集計です。期間（全期間/月間/週間）によって値が変わります。

08うまく動かないとき

  • ボタンが出ない → note.com 以外のページでは出ません。note.com を開いた状態で Tampermonkey/Userscripts のメニューを開き、スクリプトがオンになっているか確認 → オンにしたらページをリロード。
  • 「認証エラー: noteにログインしてください」と出る → 同じブラウザで note にログインし直してから、もう一度「取得」を押してください。
  • ビューが 0 ばかり／取得できない → note 側の API 仕様が変わった可能性があります。期間を変えて試し、それでもダメなら時間をおいて再度お試しください。
  • CSV を Excel で開くと文字化けする → CSV版は文字化け対策（BOM）付きで保存しています。それでも崩れる場合は「コピー」で TSV をスプレッドシートに貼る方法をお試しください。
  • iPhone で保存先がわからない → ダウンロードは「ファイル」アプリに入ります。手早く使うなら「コピー」が確実です。
