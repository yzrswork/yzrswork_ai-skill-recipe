# あそぼう！ — 3歳児向けモンテッソーリ・ミニゲーム集

iPhone Safariで3歳児が一人で遊べる、オフライン完結のブラウザゲーム集です。
全コードは `index.html` / `style.css` / `app.js` の3ファイルのみ。
外部ライブラリ・外部画像・外部フォント・CDN は一切使用していません。

## 動作確認

### ローカル（PC/Mac）
1. `project/index.html` をダブルクリック → ブラウザで開く（`file://` で動作）
2. スプラッシュ → メニューが表示される
3. 4つのカードをタップしてそれぞれのゲームを試す

### iPhone Safari
推奨方法:
1. `project/` フォルダを iCloud Drive / メール添付 / AirDrop で iPhone に転送
2. ファイルアプリから `index.html` を選択 → 「Safariで開く」
3. 共有ボタンから「ホーム画面に追加」しておくとフルスクリーンで起動できる

代替方法（同一LAN内の簡易ホスト）:
```
cd project
python3 -m http.server 8000
# iPhoneのSafariで http://<PCのIP>:8000/ を開く
```

## 設計判断（モンテッソーリ原則の実装）

### 失敗の概念を作らない
- 不正解タップは「やさしいバウンド」のみ。×印・ブブー音・赤フラッシュは一切なし
- バウンドは320ms / 6pxの控えめな揺れで、子どもが自分で誤りに気づける程度に留めた

### 自己訂正可能
- 正解は3点セットで明確化: ①光るアニメ ②上昇音階 ③パーティクル散布
- お手本の場所まで「飛んでいく」演出で、空間的にも対応関係が分かる

### 一度に一つの概念
- 各ゲームは判断軸を1つに固定（色／形／数／音）
- 例: 形ゲームでは形ごとに色が違うが、判定は「形」のみ

### 具体物から抽象へ
- 数えゲームは数字より先に「リンゴ等のフルーツ」、色ゲームは「色のついたボール」そのもの
- 文字は最小限（ひらがな表示は補助のみ）

### 集中を妨げない
- BGMなし。効果音は0.3を上限に控えめ
- 装飾アニメは正解時のみ。常時動く要素は最小限
- 広告・通知・ポップアップ皆無

### 繰り返しの自由
- 5問クリア後は花火→「もういっかい」ボタンで子どものタップ待ち（自動進行しない）
- ホームボタンは確認なしで即時メニューに戻る

## 技術仕様

| 項目 | 実装 |
|---|---|
| 入力 | `pointerdown` 統一（touch/mouse両対応・遅延ゼロ） |
| タップ領域 | 全ボタン `min-width/height: 88px` 以上、`::after`で当たり判定拡張 |
| iOSバウンス | `position: fixed` body + `overscroll-behavior: none` |
| ズーム抑止 | viewport `maximum-scale=1`、`gesturestart` preventDefault、ダブルタップ抑止 |
| 長押しメニュー | `-webkit-touch-callout: none`, `user-select: none` |
| セーフエリア | `env(safe-area-inset-*)` を body padding に適用 |
| 音 | Web Audio APIで全合成。masterGain 0.3 上限 |
| AudioContext | 初回 pointerdown で `resume()`（iOS自動再生制限対策） |
| ミュート | localStorage `asobou_muted` で永続化 |
| アニメ | `transform`/`opacity` のみ使用（layout/paint回避） |
| メモリ | 画面遷移時に `innerHTML=''` + イベントリスナー解除 + setTimeoutクリア |
| 画面回転 | `vmin`/`flex` ベース、メディアクエリで横向き調整 |
| マルチタッチ | 副指のpointerdownはpreventDefault |

## 既知の制約

- **iOSの初回サイレント**: iPhoneの物理ミュートスイッチがONの場合、Web Audioでも音が出ません（OS仕様）
- **`file://` での画像/フォント**: 本作はSVGとシステムフォントのみ使用しているため影響なし
- **Safariのフルスクリーン**: ホーム画面に追加した場合のみ完全フルスクリーン。通常のSafariタブではアドレスバーが表示されます
- **ホームインジケータ**: iPhone X以降では下端のスワイプでホームに戻る動作はOSの権限であり、Webから無効化はできません（誤操作した場合は復帰してメニューへ戻れば状態復元）
- **古い iPad / iOS 12 以前**: `aspect-ratio` CSSプロパティ未対応のため一部レイアウトが崩れる可能性があります（iOS 15+で動作確認）
