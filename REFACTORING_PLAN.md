## 2. 設計方針（リファクタリングの前提）

リファクタリングに入る前に、このリポジトリの「守るべき性質」を明文化する。

- **「1作品=1HTMLファイル」は基本的に維持する。**
  単一ファイルであることは、コピーして配れる・ブラウザで直接開ける・GitHub Pagesでビルド不要、という本リポジトリ最大の利点。共通化は「外部ファイルへの抽出」と「ファイル内の整理」のバランスを取り、ビルドツール（webpack等）は導入しない。
- **`tr808.html`（統合版）と `otamatone.html` は完全な単一ファイルのまま残す（オーナー決定）。**
  この2作品は配布対象のため、共通ファイル抽出（フェーズ2）の対象から除外する。ボイラープレートとオーディオエンジンはインラインのまま維持し、共通コードに修正が入った場合の手動反映ルールを CLAUDE.md に記す。（当初は `tr808-v2.html` が対象だったが、TR-808統合により統合版が引き継いだ）
- **`index.html` は単体で成り立つ静的な紹介ページとする（オーナー決定）。**
  外部CSS/JSやfetchに依存せず、index.html単体をどこに置いても紹介ページとして表示できる自己完結HTMLを理想形とする。したがってフェーズ2の共通CSS抽出の対象からも除外し、フェーズ4の静的再生成では「CSS込みの完全なHTML」をテンプレートから生成する。
- **iPhone Safari実機での動作が最優先。** 自動テストが無い分、各フェーズに手動検証手順を必ず含める。
- **公開URLを壊さない。** 既存ファイル名（= 公開URL）は原則変更しない。削除する場合もリダイレクトか告知を検討する。
- **1フェーズ=1PR。** 各フェーズは独立してマージ・検証・ロールバックできる粒度に保つ。

### 共通化の方式（フェーズ2・3で使用）

GitHub Pages上で動くため、共通コードは**ビルド無しで読み込める静的ファイル**として抽出する：

```
shared/
  base.css        … CSSリセット + iOSハック + safe-area
  theme.css       … --paper/--ink 等の共通パレット（テーマ別に分割可）
  audio-unlock.js … AudioContext生成＆アンロックのユーティリティ
  synth-core.js   … TR-808系共通エンジン（フェーズ3）
```

各HTMLは `<link>` / `<script src>` で読み込む。「完全な単一ファイル性」は失われるが、同一オリジン内の2〜3ファイル参照に留める。**配布対象の `tr808-v2.html` と `otamatone.html` は例外として共通化の対象外**とし、単一ファイルのまま維持する。

## 4. やらないこと（明示的なスコープ外）

過剰設計を避けるため、以下は**実施しない**と明記する：

- **TypeScript移行**（20時間超の工数に対し、この規模・用途では見合わない）
- **バンドラ/ミニファイア導入**（GitHub Pagesのビルド不要性という利点を損なう。合計300KB弱で性能問題も無い）
- **フレームワーク導入**（React等。単一HTML文化と相反する）
- **テストフレームワーク導入**（Web Audio・Canvas・タッチ操作が主体で自動テストの費用対効果が低い。smoke-test.md の手動検証で代替）
- **logs/ の整理**（作業日誌は意図的な運用。触らない）
- **既存URLの変更・リネーム**（公開済みリンクを壊さない）
