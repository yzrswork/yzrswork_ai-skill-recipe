## 経緯

静的サイト（GitHub Pages）でのPWAマネタイズ方法の調査を踏まえ、以下の結論に至った：

- **広告（AdSense等）**: ページビュー依存。電子工作ツールはリピーターが多くインプレッションが稼ぎにくい。実装コストもある
- **Amazonアフィリエイト（主役）**: 「計算した部品をそのまま購入」というUXと直結する最もナチュラルな収益化。LEDや抵抗、555タイマーICのキーワードリンクをツール出力の直下に配置することで、購入意欲が最も高いタイミングで誘導できる
- **支援リンク（note）**: 既存のnote.com/@yzrsworkへの送客。無料で実装でき、既存読者との関係強化になる
- **Proアンロック（honest model）**: BOOTHでのライセンス販売を想定。クライアント側チェックサムのみの簡易実装であり、強い保護ではない（コード内に明示注記）。コアツールは無料で価値提供し、追加機能（オペアンプ・PCBトレース幅）のみPro化する"honest-user model"を採用

## 設計

### ファイル構成
Service Workerのスコープ衝突を避けるため `bench/` サブフォルダに隔離:
- `bench/index.html` — アプリ本体（単一HTML、依存ゼロ、インラインCSS/JS）
- `bench/sw.js` — bench専用Service Worker（CACHE_NAME: bench-v1）
- `bench/manifest.webmanifest` — bench専用マニフェスト

アイコンはルートの `icons/` を `../icons/` 相対パスで再利用（新規生成なし）。

### 収益化3層
1. **Amazonアフィリエイト**: 設定モーダルでアソシエイトタグをlocalStorageに保存。各ツール出力下に「部品を探す」リンクを配置し、タグ有無で動的URL生成
2. **支援リンク**: フッターに note.com/@yzrswork へのリンク
3. **Proアンロック**: ライセンスキー（YZRS-プレフィックス + チェックサム）で2つのProツールを解除。bench:proをlocalStorageに保存

## 実装した計算ツール（無料7本 + Pro2本）

| No | ツール | 計算式 |
|---|---|---|
| 01 | オームの法則 | V=IR, P=VI=I²R=V²/R（任意2入力から残り算出） |
| 02 | LED直列抵抗 | R=(Vs-Vf)/(If/1000)、E12近似値、推奨ワット数 |
| 03 | 抵抗カラーコード | 4帯/5帯双方向変換（値→色帯 / 色帯→値） |
| 04 | 分圧回路 | Vout=Vin×R2/(R1+R2) |
| 05 | 555タイマー（アスタブル） | f=1.44/((R1+2R2)×C)、デューティ比D=(R1+R2)/(R1+2R2) |
| 06 | 電池駆動時間 | 時間=容量/電流、実用目安×0.7併記 |
| 07 | エンジニア表記パーサ | 4k7/100n/2p2等を数値変換・逆変換 |
| P1 | オペアンプ非反転ゲイン | Gain=1+Rf/Rg（Pro） |
| P2 | 基板トレース幅 | IPC-2221簡易式 外層k=0.048（Pro） |
