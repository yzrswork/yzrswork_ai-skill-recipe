# bikey 開発計画

「チャリ通」ライクな横スクロール自転車アクションゲーム。
ブラウザで開くだけで動くスタンドアロン単一HTMLとして実装する。

- 成果物: `bikey/index.html`（外部アセットなし、CDNなし、完全オフライン動作）
- 参考にした repo 内の既存作法: `virus-busters.html`（単一ファイル、canvas、日本語UI、タッチ+キーボード両対応、和紙+テラコッタ配色）

## 1. コンセプト

毎朝の通学路を自転車で駆け抜ける。障害物をジャンプとウィリーでかわし、
遅刻せずに学校へたどり着けば1日クリア。日を追うごとに道が険しくなる。

- 「n日目」制: 1ランが1日の通学。日数が進むほどスクロール速度と障害物密度が上がる
- 各日には固定のゴール距離があり、到達で「登校成功」演出 → 次の日へ
- 残機3。ぶつかると転倒して1機減、その場から少し戻って再開。全損でゲームオーバー
- ハイスコア = 到達した最高日数と累計距離。localStorage に保存

## 2. 操作

| 入力 | 動作 |
|---|---|
| Space / ↑ / 画面右半分タップ | ジャンプ（押しっぱなしで高度が伸びる可変ジャンプ） |
| ↓ / B / 画面左半分タップ長押し | ウィリー（前輪上げ。小さい障害物を踏み越えられる、地上のみ） |
| P / Esc | ポーズ |
| R（ゲームオーバー時） | リスタート |

- ジャンプは接地時のみ。コヨーテタイム約80ms + 入力先行受付（バッファ）約100msで操作感を丸くする
- ウィリー中は当たり判定の下部が甘くなり、「小」障害物（マンホール段差、縁石、小石）をノーダメージで越えられる
- タッチUIは画面下に半透明の操作ヒントを常時表示

## 3. ゲームプレイ仕様

### スクロールと物理
- 固定タイムステップ（60Hz想定、deltaTime累積方式）で更新、描画は rAF
- 自動右スクロール。基本速度は日数で上昇（day1: 5px/f 相当 → 上限 9px/f 相当）
- 重力 + 可変ジャンプ（ボタン離しで上昇打ち切り）。接地は路面y固定の平坦道路

### 障害物（すべてコードで手続き描画、画像ファイル不使用）
| 種類 | サイズ | 回避手段 |
|---|---|---|
| 縁石、マンホール段差 | 小 | ジャンプ or ウィリー |
| 歩行者（ゆっくり左へ歩く） | 中 | ジャンプ |
| 停車中の車 | 大 | ジャンプ（頭上注意の看板と組ませない） |
| 犬（速めに左へ走る） | 中 | タイミングジャンプ |
| 道路の穴（グレーチング欠落） | 幅可変 | ジャンプ。落ちたら転倒 |
| 頭上看板 | 上部 | ジャンプしない（くぐる）。ウィリーは可 |

- スポナーは「最低間隔（現在速度から逆算した反応猶予）」を保証し、理不尽な連続配置を禁止
- 日数が進むと出現テーブルに強い障害物が混ざる。組み合わせ（穴+看板など）は day3 以降

### 収集要素
- コイン（おにぎりアイコン）を空中と地上に配置。取ると+100点、10個で残機+1はやりすぎなので100個で+1機

### スコア
- 距離 1m = 10点、おにぎり +100点、ノーミスで1日クリア時ボーナス +1000点

## 4. 画面と演出

- ステージは横長 16:9（960x540 の論理解像度）、レターボックスで全画面フィット。スマホ縦持ちでは「横にしてね」表示
- 背景は3層パララックス: 空+雲 / 遠景の山と家並み / 手前の電柱と塀。すべて矩形+円の手続き描画
- 配色は repo の既存トーンに寄せる（和紙ベージュの空、テラコッタ屋根、墨色の主線）
- 自転車+ライダーは10個程度の図形パーツで描き、ペダル回転、ウィリー時の車体傾き、転倒時のくるくるアニメを付ける
- 状態遷移: タイトル → プレイ → (転倒リトライ | 日クリアジングル) → リザルト/ゲームオーバー → タイトル
- 効果音は WebAudio でオシレータ合成（ジャンプ、着地、コイン、転倒、クリアジングル）。初回タップで AudioContext を resume。ミュートボタン付き

## 5. 実装構成（単一ファイル内のモジュール分割）

```
index.html
  <style>   … レイアウト、レターボックス、UIオーバーレイ
  <canvas>  … ゲーム描画（論理960x540、devicePixelRatio対応）
  <script>
    CONFIG      … 全チューニング定数を1オブジェクトに集約
    Input       … keyboard/touch を統一した仮想ボタン(jump/wheelie/pause)
    Audio       … WebAudio SFXファクトリ
    Player      … 状態機械(ground/air/wheelie/crash)、可変ジャンプ、コヨーテ
    Spawner     … 距離ベースの障害物+コイン生成、猶予保証
    World       … パララックス背景、路面、スクロール管理
    Collision   … AABB(プレイヤーは状態で判定箱を変形)
    Game        … シーン管理(title/play/clear/gameover)、HUD、セーブ
  </script>
```

- 依存ゼロ。file:// で開いても動くこと（localStorage 不可時は try/catch でスコア保存だけ無効化）
- 全チューニング値（速度、重力、出現間隔、日数カーブ）は CONFIG に集約し、後から調整しやすくする

## 6. 受け入れ基準（実装後に検証）

1. Chromium でロードしてJSエラーゼロ
2. タイトル → 1日目開始 → ジャンプ/ウィリーで障害物回避 → 転倒 → 残機減 → ゲームオーバー → リスタート、の一連が動く
3. 1日目のゴール到達で2日目に進み、速度が上がる
4. キーボードとタッチ（タップイベント）の両方で操作できる
5. ウィンドウリサイズでレターボックスが追従する
6. ハイスコアがリロード後も残る

## 7. v2 拡張計画（縦画面対応、坂道、速度操作）

オーナー要望3点への対応。既存の受け入れ基準1-6は維持したまま拡張する。

> **方針（2026-07-02 追記）: スマホファースト。** 本作は基本的にiPhone等のスマホ縦持ちで遊ぶ前提とする。縦画面が「対応」ではなくメインの体験であり、UI/HUD/操作ヒント/ボタンサイズ/チューニングはすべて縦画面基準で設計する。横画面 + キーボードは従来どおり動作すればよい（開発検証用の位置づけ）。iOS実機での快適動作（safe-area、ズーム抑止、タッチだけで全操作完結）を受け入れ基準に含める。

### 7.1 iPhone縦画面対応

現状は論理解像度960x540固定 + 縦持ちで「横にしてね」表示。これを廃止し、縦横どちらでもプレイ可能にする。

- 論理解像度を動的化: 高さ VH=540 固定、幅 VW = clamp(round(540 * 画面アスペクト比), 300, 960)。リサイズ/回転のたびに再計算
- プレイヤーx位置は固定値190をやめ、`max(70, VW * 0.20)` で幅に比例させる
- スポーンは VW + 60 から。縦画面では視界が狭く反応猶予が減るため、**反応猶予は時間で保証する**: (スポーンx - プレイヤーx) / 現在の最大到達速度 >= 0.5秒 になるよう、狭い画面では実効最高速度（ブースト込み）に上限をかける
- HUDは VW < 480 のときコンパクト配置（フォント縮小、右上の要素を縦積み）
- rotate オーバーレイと checkOrientation を撤去
- 背景パララックスと道路描画は VW 可変で崩れないこと（タイル/ループ描画は既にworld-x基準なので幅だけ差し替え）

### 7.2 坂道（山）と坂の途中の穴

平坦路をやめ、なだらかな丘のある地形にする。

- 地形はセグメント列で生成: `flat(長さ)` と `hill(上り長, 高さ, 下り長)`。カメラ先読みで生成、通過後に破棄
- 高さは cos 補間でなめらかに接続。`groundY(worldX)` を World が提供し、全員（Player / Spawner / Collision / 描画）がこれを参照する
- 丘の高さ 40-110px、最大勾配はおよそ30度まで。day1 はほぼ平坦、day2 から低い丘、day4 以降で高い丘が混ざる
- プレイヤー: 接地中は y = groundY(プレイヤーx位置のworld x)、車体を勾配角に合わせて傾ける。空中は従来の放物線、着地判定は「y が groundY を超えたら接地」
- **勾配と速度の連動**: 上りはスクロール速度が下がり（最大 -25%）、下りは上がる（最大 +20%）。7.3の手動速度操作と掛け算で効く
- 穴: 従来の「路面の穴」を地形に埋め込む方式に変更し、**坂の途中にも配置できる**。穴区間 [x0, x1] では groundY が谷底（+30px）になり、車輪が穴区間に完全に入ったら転倒。ジャンプで飛び越える
- 障害物のy座標は決め打ち定数をやめ、スポーン時に `groundY(world x)` へ足元アンカー。看板の帯も「その地点のgroundYから見た頭上」に相対化
- 頭上看板と高い丘の頂上は組ませない（理不尽回避: 看板は勾配がほぼ0の区間にのみ置く）

### 7.3 押しながら左右スライドで減速/加速

- 速度係数 speedFactor: 0.6（最減速）から 1.5（最加速）、初期値 1.0
- **タッチ**: いずれかの指がホールド中、タッチ開始点からの横ずれ dx で操作。デッドゾーン ±15px、±130px で最大。左スライド = 減速、右スライド = 加速。指を離したら 1.0 へなめらかに復帰（約0.4秒）
- ジャンプ長押し中もウィリー長押し中も、その指のスライドで同時に速度操作できる（親指1本で完結）
- **キーボード**: ← 長押し = 減速、→ 長押し = 加速。離すと 1.0 へ復帰。タッチと同じ係数レンジ
- 実効速度 = 基本速度(日数) x speedFactor x 勾配係数。距離スコアは実効速度準拠（速く走るほど稼げるリスクリワード）
- HUDに速度メーター（小さな横バー、1.0の位置に目盛り）を追加
- タイトルとフッターの操作ヒントを更新

### 7.4 v2 受け入れ基準

1. iPhone縦相当（390x844, タッチ）でrotate表示が出ず、プレイ開始から転倒までエラーなく動く
2. 横画面でも従来どおり16:9相当で動く。リサイズで論理幅が追従する
3. 丘を上り下りでき、坂の途中の穴をジャンプで越えられる。穴に入ると転倒する
4. 坂上で自然に減速、坂下で加速する
5. タッチのスライドとキーボード ←/→ の両方で減速/加速でき、離すと1.0へ戻る
6. 既存基準1-6（v1）も引き続き満たす
